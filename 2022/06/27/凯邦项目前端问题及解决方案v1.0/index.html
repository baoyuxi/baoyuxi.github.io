7.3.弹窗

  • 问题：弹窗出现$refs.alertDialog.open()中的open为undefined

  • 原因： uni-app中的组件有些为独立组件，不可包含于其他组件内，最好单独写于模块外，写于模块内有时也会出现此问题

八、父子组件传值（数据不刷新）

  • 父组件向子组件传值，将dataMsg传递给子组件

  • 子组件props接收dataMsg，watch监听dataMsg的变化

  • 其次vue中我们会遇到很多父子组件通信的需求，下面简单列一下，父子组件通信的几种情况
      1. 父组件向子组件传值：使用prop向子组件传值；
      2. 子组件实时监听父组件传来的值的变化：使用watch去监听父组件传来的值；
      3. 子组件可以通过this.$parent.去访问父组件的值或方法；

九、表单校验

9.1 校验消息无法消除

  • 问题：对于表单内文档是否上传进行验证，只添加于表单验证规则中，会出现一直显示校验消息

  • 解决方法：对于文档或者图片是否上传验证，需在上传成功后用clearValidate移除校验结果

9.2 常见易错点

    1. el-from是否绑定了值model
    2. el-from是否绑定了验证规则：rules
    3. el-from-item标签是否有prop,并且prop的值是否和rules的值对应
    4. el-from-item标签中prop的名称和el-input标签中v-model的名称一致

十、下拉刷新

  • 采用下拉刷新获取数据列表，请求完最后一页的数据后，不小心又进行了一次下拉操作，相当于又重新请求了一次

    • 解决方法：可以通过

      在onLoad中（即页面刚加载进来时）调用该API实现开始下拉功能；在下拉刷新函数中，调用后台接口获取每一页的数据，将每次获取的数据列表累加起来，直到请求到最后一页数据后，调用

      关闭下拉刷新，这时，即便再次下拉，也不会请求到接口数据了。或者在onPullDownRefresh()调用时将列表清空

十一、函数方法

11.1.toFixed函数

  • 问题：toFixed函数对数据处理时，在插值中直接对数据进行处理，实现结果没问题，但后台打印会出现toFixed is not a function
  • 解决方法：toFixed只能针对Number类型才能使用，所以对于字符类型的要用parseFloat或者parseInt函数先转一下再调用

11.2.indexOf方法

  • 问题： 用indexOf方法在uni-app中报错，运用在template中出现undefined情况
  • 原因：
    1. 由于indexOf，检索字符串值时必须要有的，如果检索字符串值没有出现，则该方法将会返回-1
    2. 出现undefined的原因，是没有去处理返回-1的情况
